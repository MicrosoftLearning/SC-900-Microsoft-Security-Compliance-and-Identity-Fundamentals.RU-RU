# Ролик. Microsoft Sentinel

Эта демонстрация выполняется на основе следующего содержимого Learn:

- Схема обучения. Описание возможностей решений безопасности Майкрософт
- Модуль. Описание возможностей обеспечения безопасности Microsoft Sentinel
- Урок. Описание возможностей обнаружения и устранения угроз в Microsoft Sentinel

## Сценарий демонстрации

В этой демонстрации вы изучите некоторые варианты, доступные в Microsoft Sentinel, включая использование центра содержимого для поиска упакованных решений, которые можно развернуть.  Но сначала вы узнаете, как настроить разрешения на управление доступом на основе ролей для пользователей, которым потребуется доступ к ресурсам Microsoft Sentinel.

### Демонстрация, часть 1.

Экземпляр Microsoft Sentinel должен быть уже создан в рамках предварительной демонстрационной настройки. Убедитесь, что он создан.

1. Откройте вкладку браузера **Главная — Microsoft Azure**.  Если вы ранее закрыли вкладку, откройте страницу браузера и в адресной строке введите **https://portal.azure.com**. Войдите с учетными данными Azure, предоставленными авторизованным организатором лаборатории (ALH).  Откроется домашняя страница служб Azure.

1. В поле поиска, которое расположено на синей верхней панели рядом с надписью Microsoft Azure, введите **Microsoft Sentinel**, а затем выберите **Microsoft Sentinel** в результатах поиска.  

1. На странице Microsoft Sentinel вы увидите свой экземпляр Sentinel в списке и выберите его.  Если его нет в списке, создайте его сейчас.
    1. На странице Microsoft Sentinel выберите **Создать Microsoft Sentinel**.

    1. На странице "Добавить Microsoft Sentinel в рабочую область" выберите элемент **Создать рабочую область**. На вкладке «Основные» рабочей области страницы «Создание рабочей области Log Analytics» введите следующее:
        1. Подписка: оставьте значение по умолчанию.
        1. Группа ресурсов: выберите **Создать**, затем введите имя **SC900-Sentinel-RG** и нажмите кнопку **OK**.
        1. Имя: **SC900-LogAnalytics-workspace**.
        1. Регион: **Восточная часть США** (другой регион по умолчанию можно выбрать в зависимости от вашего расположения)
        1. Выберите **Проверка и создание** (теги не будут настроены).
        1. Проверьте введенные сведения, затем выберите **Создать**.
        1. Появление новой рабочей области в списке может произойти через одну – две минуты. Если эта новая область не отображается, выберите **Обновить**, затем нажмите **Добавить**.
        1. После добавления новой рабочей области отобразится страница "Microsoft Sentinel | Новости и руководства", на которой указано, что активирована бесплатная пробная версия Microsoft Sentinel.  Щелкните **ОК**.

1. Не закрывайте эту страницу, так как она будет использоваться в следующей задаче.

### Демонстрация, часть 2.

Как и в случае со всеми ресурсами Azure, необходимо убедиться, что у пользователей есть соответствующие разрешения на доступ к ресурсам Microsoft Sentinel. Здесь вы увидите шаги по назначению роли и доступные роли для использования с Microsoft Sentinel.  

1. В поле поиска, которое расположено на синей верхней панели рядом с надписью Microsoft Azure, введите **группы ресурсов**, затем выберите **Группы ресурсов** в результатах поиска. Назначение роли на уровне группы ресурсов обеспечит применение этой роли ко всем ресурсам, развернутым для поддержки Microsoft Sentinel.

1. На странице групп ресурсов выберите группу ресурсов, созданную с помощью Microsoft Sentinel, с именем **SC900-Sentinel-RG**.

1. На странице SC900-Sentinel-RG выберите **Управление доступом (IAM)** на панели навигации слева.

1. На странице управления доступом выберите **Просмотреть мой доступ**.  Если вы используете подписку "Квалифицированный облачный срез", для назначения роли устанавливается значение ВЛАДЕЛЕЦ LOD, которое является настраиваемым назначением роли, настроенным для этой подписки на облачный срез, и предоставляет необходимые разрешения. Однако для демонстрационных целей рекомендуется показать конкретные роли Sentinel.  Закройте окно назначений, щелкнув значок **X** в его правом верхнем углу.

    1. На странице управления доступом выберите **+Добавить**, затем выберите **Добавить назначение ролей**.

    1. Откроется окно «Добавить назначение ролей».  В поле поиска введите **Microsoft Sentinel** , чтобы просмотреть роли, связанные с Microsoft Sentinel.
    1. Для любой из указанных ролей выберите **Просмотреть**, чтобы просмотреть сведения о ней.  Рекомендуется назначить наименьшую привилегию, необходимую для этой роли.  

    1. Закройте это окно, щелкнув значок **X** в правом верхнем углу экрана.

1. На странице управления доступом закройте это окно, щелкнув значок **X** в правом верхнем углу окна.

### Демонстрация, часть 3.

В этой части демонстрации вы покажете шаги по подключению к источнику данных. Многие соединители данных развертываются как часть решения Microsoft Sentinel вместе со связанным содержимым, таким как правила аналитики, книги и сборники схем. Центр содержимого Microsoft Sentinel — это централизованное расположение для обнаружения встроенного содержимого и управления им. На этом шаге вы будете использовать центр содержимого для развертывания решения Microsoft Defender for Cloud для Microsoft Sentinel.  Это решение позволяет принимать оповещения системы безопасности, сообщаемые в Microsoft Defender для облака.

1. Откройте вкладку браузера для Microsoft Sentinel.

1. На панели навигации слева выберите **Центр содержимого**.

1. Прокрутите вниз, чтобы просмотреть длинный список доступных решений и параметры для фильтрации списка.  В этой демонстрации мы ищем **Microsoft Defender для облака**.  и выберите ее в списке.  В открывавшемся боковом окне прочтите описание и выберите **Установить**.  Это решение включает один соединитель данных и одно правило аналитики. Установка может занять минуту.  Щелкните **Управление**.

1. Установите поле рядом с надписью Microsoft Defender для облака.  В правой части страницы откроется окно.  Выберите **Открыть страницу соединителя**.

1. Обратите внимание на инструкции по настройке.  Установите флажок рядом с именем подписки, а затем выберите **Подключиться**.  Состояние переместится на подключено.  Соединитель теперь включен, хотя его отображение на странице соединителей данных может занять некоторое время.  

1. Теперь просмотрите сведения о правиле аналитики.  В верхней части страницы (в области навигации) выберите **Microsoft Defender для облака**.  Установите флажок рядом с надписью "Detect CoreBackUp Deletion Activity from related security alerts" (Обнаружение действия удаления CoreBackUp из связанных оповещений системы безопасности). В открываемом окне вы увидите сведения о правиле и его действиях.  Вы можете выполнить действия по настройке правила.  **ПРИМЕЧАНИЕ**. Сведения о логике правила выходят за рамки область основных принципов, но вы можете показать тип сведений, которые можно настроить как часть правила.  
    1. Щелкните **Конфигурация**.
    1. Выберите правило **Обнаружение действия удаления CoreBackUp из связанных оповещений системы безопасности**.
    1. В окне, которое откроется в правой части страницы, выберите **Создать правило**.
    1. Перейдите по каждой из страниц конфигурации, кратко и высокоуровнево, затем выберите **Проверить и создать**, а затем **нажмите кнопку Сохранить**.

1. Вернитесь на страницу Sentinel, выбрав **Microsoft Sentinel | Центр содержимого** в верхней части страницы, где написано правила аналитики.

1. Обратитесь к тому, что с помощью центра содержимого можно легко и быстро развернуть решение.

1. Оставьте эту страницу открытой, так как она будет использоваться в следующей задаче.

### Демонстрация, часть 4.

В этой части демонстрации вы покажете некоторые параметры, доступные в Sentinel.

1. На панели навигации слева выберите **Охота**.  В верхней части страницы выберите вкладку **Запросы** . Ознакомьтесь с описанием охотничьего запроса. Запросы охоты можно добавлять через центр содержимого. Все ранее установленные запросы будут перечислены здесь. Выберите **Перейти к центру содержимого**.  Центр содержимого содержит содержимое, включающее запросы, как часть решения или как автономный запрос.  Прокрутите вниз, чтобы просмотреть доступные параметры.

1. На панели навигации слева выберите **MITRE ATT&CK**.  MITRE ATT&CK — это общедоступная база знаний тактик и методов, которые часто используются злоумышленниками. С помощью Microsoft Sentinel вы можете просматривать обнаружения, которые уже активны в рабочей области и доступны для настройки, чтобы лучше понимать обеспечение безопасности в вашей организации на основе тактик и методов платформы MITRE ATT&CK®.  Выделите любую ячейку таблицы и запишите сведения, доступные в правой части экрана.  

1. В области навигации слева выберите **Сообщество**. Аналитики безопасности Майкрософт постоянно создают и добавляют новые книги, сборники схем, запросы для поиска угроз и многое другое и публикуют все это в сообществе, чтобы вы могли использовать эти материалы в вашей среде. Вы можете загрузить пример содержимого из частного сообщества репозитория GitHub, чтобы создавать настраиваемые книги, запросы поиска угроз, записные книжки и сборники схем для Microsoft Sentinel.  Выберите **Подключить контент сообщества**.  Откроется новая вкладка в репозитории GitHub, где можно скачать содержимое для поддержки сценариев.  Вернитесь на вкладку Azure в браузере.

1. На панели навигации слева выберите **Аналитика**.  Должно быть два активных правила: одно доступно по умолчанию, и правило, созданное в предыдущей задаче. Выберите правило по умолчанию **Расширенное обнаружение многоэтапных атак**.  Запишите подробные сведения.  Microsoft Sentinel использует Fusion (подсистему корреляции на основе масштабируемых алгоритмов машинного обучения) для автоматического обнаружения многоступенчатых атак (также называются постоянными серьезными угрозами), определяя сочетания аномальных и подозрительных действий на различных этапах атаки. На основе этих наблюдений в Microsoft Sentinel создаются инциденты, которые трудно выявить иным образом.

1. На панели навигации слева выберите **Автоматизация**.  Здесь можно создавать простые правила автоматизации, осуществлять интеграцию с существующими сборниками схем или создавать новые.  Выберите **+ Создать** и **Правило автоматизации**.  Обратите внимание на окно, которое открылось в правой части экрана, и доступные варианты для создания условий и действий.  Выберите **Отмена** в нижней части экрана.

1. На панели навигации слева выберите **Книги**. Ознакомьтесь с описанием книги Microsoft Sentinel.  Книги можно добавлять через центр содержимого. Здесь будут перечислены все ранее установленные книги. Выберите **Перейти к центру содержимого**.  Центр содержимого содержит содержимое, включающее книги как часть решения или как автономную книгу. Прокрутите вниз, чтобы просмотреть доступные параметры.

1. Закройте это окно, щелкнув значок **X** в правом верхнем углу экрана.

1. В левом верхнем углу окна под синей панелью выберите **Главная**, чтобы вернуться на домашнюю страницу портала Azure.  

### Просмотр

В этой демонстрации вы выполнили инструкции по подключению Microsoft Sentinel к источникам данных и настройке книги, а также изучили несколько параметров, доступных в Microsoft Sentinel.
