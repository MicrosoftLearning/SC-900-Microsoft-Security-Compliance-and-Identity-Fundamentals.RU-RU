# <a name="demo-microsoft-sentinel"></a>Ролик. Microsoft Sentinel

Эта демонстрация выполняется на основе следующего содержимого Learn:

- Схема обучения. Описание возможностей решений безопасности Майкрософт
- Модуль. Описание возможностей обеспечения безопасности Microsoft Sentinel
- Урок. Описание принципов встроенного управления угрозами в Microsoft Sentinel

## <a name="demo-scenario"></a>Сценарий демонстрации

В рамках этой демонстрации вы рассмотрите процесс создания экземпляра Microsoft Sentinel.  Вы также настроите разрешения, чтобы обеспечить доступ к ресурсам, которые будут развернуты для поддержки Microsoft Azure Sentinel.  После завершения базовой настройки вы выполните действия по подключению Microsoft Sentinel к источникам данных и выберите книгу для мониторинга и визуализации данных.  Наконец, вы покажете некоторые другие доступные варианты, включая встроенную аналитику для получения уведомлений о подозрительных действиях, возможности автоматизации и т. д.

### <a name="pre-demo-setup--create-a-microsoft-sentinel-instance"></a>Настройка перед демонстрацией. Создание экземпляра Microsoft Sentinel

1. Откройте вкладку браузера **Главная — Microsoft Azure**.  Если ранее вы закрыли эту вкладку, откройте страницу браузера и в адресной строке введите portal.azure.com, затем выполните вход.

1. В поле поиска, которое расположено на синей верхней панели рядом с надписью Microsoft Azure, введите **Microsoft Sentinel**, а затем выберите **Microsoft Sentinel** в результатах поиска.

1. На странице Microsoft Sentinel выберите **Создать Microsoft Sentinel**.

1. На странице "Добавить Microsoft Sentinel в рабочую область" выберите элемент **Создать рабочую область**.

1. На вкладке «Основные» рабочей области страницы «Создание рабочей области Log Analytics» введите следующее:
    1. Подписка: оставьте значение по умолчанию.
    1. Группа ресурсов: выберите **Создать**, затем введите имя **SC900-Sentinel-RG** и нажмите кнопку **OK**.
    1. Имя: **SC900-LogAnalytics-workspace**.
    1. Регион: **Восточная часть США** (другой регион по умолчанию можно выбрать в зависимости от вашего расположения)
    1. Выберите **Проверка и создание** (теги не будут настроены).
    1. Проверьте введенные сведения, затем выберите **Создать**.
    1. Появление новой рабочей области в списке может произойти через одну – две минуты. Если эта новая область не отображается, выберите **Обновить**, затем нажмите **Добавить**.

1. После добавления новой рабочей области отобразится страница "Microsoft Sentinel | Новости и руководства", на которой указано, что активирована бесплатная пробная версия Microsoft Sentinel.  Щелкните **ОК**.  Обратите внимание на три действия, приведенные на странице «Начало работы».

1. Оставьте эту страницу открытой, так как она будет использоваться в следующей задаче.

### <a name="demo-part-2"></a>Демонстрация, часть 2.

При создании экземпляра Microsoft Sentinel необходимо убедиться в том, что у вас есть необходимый доступ к ресурсам, которые развертываются для поддержки Microsoft Sentinel.  

1. В поле поиска, которое расположено на синей верхней панели рядом с надписью Microsoft Azure, введите **группы ресурсов**, затем выберите **Группы ресурсов** в результатах поиска. Назначение роли на уровне группы ресурсов обеспечит применение этой роли ко всем ресурсам, развернутым для поддержки Microsoft Sentinel.

1. На странице групп ресурсов выберите группу ресурсов, созданную с помощью Microsoft Sentinel, с именем **SC900-Sentinel-RG**.

1. На странице SC900-Sentinel-RG выберите **Управление доступом (IAM)** на панели навигации слева.

1. На странице управления доступом выберите **Просмотреть мой доступ**.  Как у администратора MOD вашей текущей ролью будет "администратор службы".  Это предоставит вам необходимые разрешения, но для целей демонстрации может потребоваться отобразить определенные роли Sentinel.  Закройте окно назначений администратора MOD, выбрав **X** в правом верхнем углу окна.

    1. На странице управления доступом выберите **+Добавить**, затем выберите **Добавить назначение ролей**.

    1. Откроется окно «Добавить назначение ролей».  В поле поиска роли введите **Microsoft Sentinel** для просмотра 4 ролей, связанных с Microsoft Sentinel.
    1. Для любой из указанных ролей выберите **Просмотреть**, чтобы просмотреть сведения о ней.  Рекомендуется назначить наименьшую привилегию, необходимую для этой роли.  

    1. Закройте это окно, щелкнув значок **X** в правом верхнем углу экрана.

1. На странице управления доступом закройте это окно, щелкнув значок **X** в правом верхнем углу окна.

### <a name="demo-part-3"></a>Демонстрация, часть 3.

В этой части демонстрации вы покажете шаги по подключению к источнику данных.  В частности, вы подключитесь к соединителю данных Microsoft Defender для облака.

1. В поле поиска, которое расположено на синей верхней панели рядом с надписью Microsoft Azure, введите **Microsoft Sentinel**, а затем выберите **Microsoft Sentinel** в результатах поиска.

1. На странице Microsoft Sentinel выберите созданную рабочую область с экземпляром Microsoft Sentinel с именем **SC900-LogAnalytics-workspace**.

1. Первый шаг работы с Microsoft Sentinel — получить возможность сбора данных. На панели навигации слева выберите пункт **Соединители данных**, который отображается под конфигурацией.

1. На странице «Соединители данных» прокрутите до главного окна, чтобы просмотреть расширенный список доступных соединителей. В поле поиска на странице соединителей данных введите **Microsoft Defender для облака**, а затем в списке выберите пункт **Microsoft Defender для облака**.

1. Откроется окно соединителя Microsoft Defender для облака. Просмотрите описание и выберите **Открыть страницу соединителя**.

1. На странице соединителя Microsoft Defender для облака просмотрите описание в левой части окна.

1. На вкладке инструкций в главном окне приводятся предварительные требования.  Просмотрите инструкции и сведения о настройке.
    На вкладке инструкций в главном окне приводятся предварительные требования.  Просмотрите инструкции и сведения о настройке.
    1. В разделе конфигурации выберите пустое поле рядом с подпиской **MOC Subscription--lodXXXXXXXX** в списке, чтобы в синем поле появился флажок, а затем выберите команду **Подключиться** (находится над полем поиска).  Откроется окно "Подключение". Нажмите кнопку **ОК**.  В столбце состояния рядом с подпиской вы увидите, что состояние изменилось на "Подключено".  Если состояние "Подключено" не отображается в окне в левой части страницы, ничего страшного. Не обновляйте содержимое в браузере.
    1. Прокрутите страницу вниз и выберите команду **Включить**, чтобы инциденты создавались автоматически на основе всех оповещений, созданных в подключенной службе.
    1. Теперь выберите вкладку **Дальнейшие действия** в верхней части страницы, чтобы просмотреть рекомендуемые книги для этого соединителя данных.  В Microsoft Sentinel доступны встроенные шаблоны книг, которые позволяют быстро извлечь аналитические сведения из существующих данных сразу после подключения к источнику данных.
    1. Выберите **Соответствие требованиям и защита ASC** (обратите внимание, что ASC или Центр безопасности Azure теперь называется Microsoft Defender для облака).  Откроется страница книг.  В правой части экрана просмотрите описание и нажмите кнопку **Сохранить** в нижней части экрана, а затем нажмите кнопку **ОК**, чтобы сохранить книгу в расположении по умолчанию.  Теперь выберите **Просмотреть сохраненную книгу**.  
    1. В поле рабочей области выберите **SC900-LogAnalytics-workspace**.
    1. В верхней части страницы книги выберите **Автоматическое обновление: выкл.** , а затем выберите значение **5 минут** и нажмите кнопку **Применить**.
    1. В верхней части страницы книги щелкните значок **Сохранить**.
    1. В левом верхнем углу страницы "Книги" выберите **Microsoft Sentinel** над надписью "Книги". Вы вернетесь на страницу обзора. Теперь вы должны увидеть выше число 1 рядом с состоянием "Подключено". Это означает, что есть один активный соединитель (может потребоваться обновить страницу).

1. Оставьте эту страницу открытой, так как она будет использоваться в следующей задаче.

### <a name="demo-part-4"></a>Демонстрация, часть 4.

В этой части демонстрации вы покажете некоторые параметры, доступные в Sentinel.

1. На панели навигации слева выберите **Охота**.  На выбранной вкладке **Запросы** (подчеркнута), выберите любой запрос из списка.  После выбора запроса обратите внимание на сведения, предоставленные об этом запросе, включая код запроса, а также параметр для выполнения запроса и просмотра результатов.  Не выбирайте ничего.

1. На панели навигации слева выберите **MITRE ATT&CK**.  MITRE ATT&CK — это общедоступная база знаний тактик и методов, которые часто используются злоумышленниками. С помощью Microsoft Sentinel вы можете просматривать обнаружения, которые уже активны в рабочей области и доступны для настройки, чтобы лучше понимать обеспечение безопасности в вашей организации на основе тактик и методов платформы MITRE ATT&CK®.  Выделите любую ячейку таблицы и запишите сведения, доступные в правой части экрана.  

1. В области навигации слева выберите **Сообщество**. Аналитики безопасности Майкрософт постоянно создают и добавляют новые книги, сборники схем, запросы для поиска угроз и многое другое и публикуют все это в сообществе, чтобы вы могли использовать эти материалы в вашей среде. Вы можете загрузить пример содержимого из частного сообщества репозитория GitHub, чтобы создавать настраиваемые книги, запросы поиска угроз, записные книжки и сборники схем для Microsoft Sentinel.  Выберите **Подключить контент сообщества**.  Откроется новая вкладка в репозитории GitHub, где можно скачать содержимое для поддержки сценариев.  Вернитесь на вкладку Azure в браузере.

1. На панели навигации слева выберите **Аналитика**.  Выберите первый элемент из списка **Расширенное обнаружение многоступенчатых атак**.  Запишите подробные сведения.  Microsoft Sentinel использует Fusion (подсистему корреляции на основе масштабируемых алгоритмов машинного обучения) для автоматического обнаружения многоступенчатых атак (также называются постоянными серьезными угрозами), определяя сочетания аномальных и подозрительных действий на различных этапах атаки. На основе этих наблюдений в Microsoft Sentinel создаются инциденты, которые трудно выявить иным образом.

1. На панели навигации слева выберите **Автоматизация**.  Здесь можно создавать правила автоматизации, осуществлять интеграцию с существующими сборниками схем или создавать новые.  Выберите **+ Создать** и **Правило автоматизации**.  Обратите внимание на окно, которое открылось в правой части экрана, и доступные варианты для создания условий и действий.  Выберите **Отмена** в нижней части экрана.

1. На панели навигации слева выберите **Книги**. На странице «Книги» выберите вкладку **Мои книги** над полем поиска.  Сохраненная ранее книга появится в списке и будет доступна для просмотра и мониторинга ваших данных.   Примечание. В подписке Azure нет никаких реальных действий для отражения в книге, а в подписках на лабораторную среду Azure при сборе данных, которые можно визуализировать в книге, могут возникать задержки дольше обычных.

1. Закройте это окно, щелкнув значок **X** в правом верхнем углу экрана.

1. В левом верхнем углу окна под синей панелью выберите **Главная**, чтобы вернуться на домашнюю страницу портала Azure.
