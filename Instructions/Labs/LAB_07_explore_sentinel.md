# Практическое занятие. Обзор Microsoft Sentinel

Это практическое занятие выполняется на основе следующего содержимого Learn:

- Схема обучения. Описание возможностей решений безопасности Майкрософт
- Модуль. Описание возможностей обеспечения безопасности Microsoft Sentinel
- Урок. Описание возможностей обнаружения и устранения угроз в Microsoft Sentinel

## Сценарий практического занятия

Вы рассмотрите процесс создания экземпляра Microsoft Sentinel.  Вы также настроите разрешения, чтобы обеспечить доступ к ресурсам, которые будут развернуты для поддержки Microsoft Sentinel.  После выполнения этой базовой настройки вы выполните действия по подключению Microsoft Sentinel к источникам данных и настройке книги, а также кратко изучите некоторые ключевые возможности Microsoft Sentinel.

**Примерное время**: 45–60 мин

### Задача 1

Создание экземпляра Microsoft Sentinel

1. Вы должны быть на домашней странице служб Azure.  Если вы ранее закрыли браузер, откройте Microsoft Edge. В адресной строке введите **portal.azure.com** и войдите с учетными данными администратора.

1. В синем поле поиска в верхней части страницы введите **Microsoft Sentinel**, а затем выберите **Microsoft Sentinel** в результатах поиска.

1. На странице Microsoft Sentinel выберите **Создать Microsoft Sentinel**.

1. На странице "Добавить Microsoft Sentinel в рабочую область" выберите элемент **Создать рабочую область**.

1. На вкладке «Основные» рабочей области страницы «Создание рабочей области Log Analytics» введите следующее:
    1. Подписка: оставьте значение по умолчанию; это подписка Azure, предоставляемая авторизованным поставщиком услуг размещения практических занятий (ALH).
    1. Группа ресурсов: выберите **SC900-Sentinel-RG**. Если эта группа ресурсов отсутствует в списке, создайте ее, выбрав **Создать,** введите **SC900-Sentinel-RG** и нажмите **кнопку ОК**.
    1. Имя: **SC900-LogAnalytics-workspace**.
    1. Регион: **Восточная часть США** (другой регион по умолчанию можно выбрать в зависимости от вашего расположения)
    1. Выберите **Проверка и создание** (теги не будут настроены).
    1. Проверьте введенные сведения, затем выберите **Создать**.
    1. Появление новой рабочей области в списке может произойти через одну – две минуты. Если эта новая область не отображается, выберите **Обновить**, затем нажмите **Добавить**.

1. После добавления новой рабочей области отобразится страница "Microsoft Sentinel | Новости и руководства", на которой указано, что активирована бесплатная пробная версия Microsoft Sentinel.  Щелкните **ОК**.  Обратите внимание на три действия, приведенные на странице «Начало работы».

1. Оставьте эту страницу открытой, так как она будет использоваться в следующей задаче.

### Задача 2

После создания экземпляра Microsoft Sentinel важно, чтобы пользователи, которые будут отвечать за поддержку Microsoft Sentinel, имели необходимые разрешения.  Для этого соответствующим пользователям назначаются необходимые разрешения роли.  В этой задаче вы ознакомитесь с доступными встроенными ролями Microsoft Sentinel.

1. В синем поле поиска введите **группы ресурсов**, а затем в результатах поиска выберите **Группы ресурсов**. 

1. На странице групп ресурсов выберите группу ресурсов, созданную с помощью Microsoft Sentinel, с именем **SC900-Sentinel-RG**.  Работа на уровне группы ресурсов гарантирует, что выбранная роль будет применяться ко всем ресурсам, которые входят в экземпляр Microsoft Sentinel, созданный в предыдущей задаче.

1. На странице SC900-Sentinel-RG выберите **Управление доступом (IAM)** на панели навигации слева.

1. На странице управления доступом выберите **Просмотреть мой доступ**.  Для подписки Azure, предоставленной авторизованным поставщиком услуг размещения практических занятий, определена роль, которая дает вам доступ к управлению всеми необходимыми ресурсами, как показано в описании. Однако важно разбираться в имеющихся в Sentinel ролях.  Закройте окно назначений, щелкнув значок **X** в его правом верхнем углу.

1. На странице "Управление доступом" выберите вкладку **Роли** в верхней части страницы.
    1. В поле поиска введите **Microsoft Sentinel** для просмотра встроенных ролей, связанных с Microsoft Sentinel.
    1. Для любой из указанных ролей выберите **Просмотреть**, чтобы просмотреть сведения о ней.  Рекомендуется назначить наименьшую привилегию, необходимую для этой роли.  
    1. Закройте это окно, щелкнув значок **X** в правом верхнем углу экрана.

1. На странице управления доступом закройте это окно, щелкнув значок **X** в правом верхнем углу окна.

1. В левом верхнем углу окна, под синей панелью, где указано Microsoft Azure, выберите **Главная** , чтобы вернуться на домашнюю страницу служб Azure.

1. Не закрывайте вкладку Azure в браузере.

### Задача 3

Цель этой задачи — пошаговое руководство по подключению к источнику данных. Многие соединители данных развертываются как часть решения Microsoft Sentinel вместе со связанным содержимым, таким как правила аналитики, книги и сборники схем. Центр содержимого Microsoft Sentinel — это централизованное расположение для обнаружения встроенного содержимого и управления ими. На этом шаге вы будете использовать центр содержимого для развертывания решения Microsoft Defender для облака для Microsoft Sentinel.  Это решение позволяет принимать оповещения системы безопасности, сообщаемые в Microsoft Defender для облака.

1. На домашней странице служб Azure выберите Microsoft Sentinel, а затем выберите созданный экземпляр **SC900-LogAnalytics-workspace**.

1. На панели навигации слева выберите **Центр содержимого**.

1. Прокрутите вниз, чтобы просмотреть длинный список доступных решений и параметры для фильтрации списка.  Для этой задачи вы ищете **Microsoft Defender для облака**.  и выберите ее в списке.  В открывавшемся боковом окне прочтите описание и выберите **Установить**.  Это решение включает в себя один соединитель данных и одно правило аналитики. Установка может занять минуту.  Щелкните **Управление**.

1. Выберите поле рядом с надписью Microsoft Defender для облака.  В правой части страницы откроется окно.  Выберите **Открыть страницу соединителя**.

1. Обратите внимание на инструкции по настройке.  Установите флажок рядом с именем подписки, а затем нажмите **кнопку Подключить**.  Может появиться всплывающее окно, указывающее, что потоковую передачу Microsoft Defender для облачных оповещений будут запускать только подписки, для которых у вас есть разрешения читателя безопасности.  Щелкните **ОК**.  Состояние будет перемещено на подключено.  Соединитель теперь включен, хотя для его отображения на странице соединителей данных может потребоваться некоторое время.  

1. Теперь просмотрите сведения о правиле аналитики.  В верхней части страницы (в области навигации) выберите **Microsoft Defender для облака**. Снимите флажок рядом с надписью Microsoft Defender для облака, так как соединитель уже настроен (для исчезновения значка предупреждения может потребоваться некоторое время). Выберите поле рядом с надписью **Обнаружение действия удаления CoreBackUp из связанных оповещений системы безопасности**. В открывавшемся окне вы увидите сведения о правиле и его действиях.  
    1. Хотя сведения о логике правила выходят за рамки область основных принципов, выполните действия по настройке правила, чтобы просмотреть тип сведений, которые можно настроить как часть правила. Щелкните **Конфигурация**.
    1. Выберите правило **Обнаружение действия удаления CoreBackUp из связанных оповещений системы безопасности**.
    1. В окне, которое откроется в правой части страницы, выберите **Создать правило**.
    1. Перейдите по каждой из страниц конфигурации, выберите **Просмотр и создание**, а затем нажмите **кнопку Сохранить**.

1. Вернитесь на страницу Sentinel, выбрав **Microsoft Sentinel | Центр содержимого** из хлебной крошки в верхней части страницы, над которой отображаются правила аналитики.

1. Оставьте эту страницу открытой, так как она будет использоваться в следующей задаче.


### Задача 4.

В этой задаче описаны некоторые параметры, доступные в Sentinel.

1. На панели навигации слева выберите **Охота**.  В верхней части страницы выберите вкладку **Запросы** . Ознакомьтесь с описанием запроса охоты. Запросы охоты можно добавить через центр содержимого. Здесь будут перечислены все ранее установленные запросы. Выберите **Перейти к центру содержимого**.  В центре содержимого содержится содержимое, включающее запросы как часть решения, так и как автономный запрос.  Прокрутите вниз, чтобы просмотреть доступные параметры. Закройте центр содержимого, щелкнув **X** в правом верхнем углу окна.

1. На панели навигации слева выберите **MITRE ATT&CK**.  MITRE ATT&CK — это общедоступная база знаний тактик и методов, которые часто используются злоумышленниками. С помощью Microsoft Sentinel вы можете просматривать обнаружения, которые уже активны в рабочей области и доступны для настройки, чтобы лучше понимать обеспечение безопасности в вашей организации на основе тактик и методов платформы MITRE ATT&CK®.  Выделите любую ячейку таблицы и запишите сведения, доступные в правой части экрана. **Примечание**. Для просмотра информационной панели может потребоваться выбрать "**<<**" в крайней правой части окна.

1. В области навигации слева выберите **Сообщество**. Аналитики безопасности Майкрософт постоянно создают и добавляют новые книги, сборники схем, запросы для поиска угроз и многое другое и публикуют все это в сообществе, чтобы вы могли использовать эти материалы в вашей среде. В правой части экрана выберите **Подключить контент сообщества**.  Откроется новая вкладка в репозитории GitHub, где можно скачать содержимое для поддержки сценариев. Прокрутите содержимое вниз до раздела README.md и просмотрите описание. Вернитесь на вкладку Azure в браузере.

1. На панели навигации слева выберите **Аналитика**.  Должно быть два активных правила: одно доступно по умолчанию, и правило, созданное в предыдущей задаче. Выберите правило по умолчанию **Расширенное многоэтапное обнаружение атак**.  Запишите подробные сведения.  Microsoft Sentinel использует Fusion (подсистему корреляции на основе масштабируемых алгоритмов машинного обучения) для автоматического обнаружения многоступенчатых атак (также называются постоянными серьезными угрозами), определяя сочетания аномальных и подозрительных действий на различных этапах атаки. На основе этих наблюдений в Microsoft Sentinel создаются инциденты, которые трудно выявить иным образом. **Примечание**. Для просмотра информационной панели может потребоваться выбрать "**<<**" в крайней правой части окна.

1. На панели навигации слева выберите **Автоматизация**.  Здесь можно создавать простые правила автоматизации, осуществлять интеграцию с существующими сборниками схем или создавать новые.  Выберите **+ Создать** и **Правило автоматизации**.  Обратите внимание на окно, которое открылось в правой части экрана, и доступные варианты для создания условий и действий.  Выберите **Отмена** в нижней части экрана.

1. На панели навигации слева выберите **Книги**. Ознакомьтесь с описанием книги Microsoft Sentinel.  Книги можно добавлять через центр содержимого. Все ранее установленные книги будут перечислены здесь. Выберите **Перейти к центру содержимого**.  Центр содержимого содержит содержимое, включающее книги, в составе решения или в виде автономной книги. Прокрутите вниз, чтобы просмотреть доступные параметры.

1. Закройте это окно, щелкнув значок **X** в правом верхнем углу экрана.

1. В левом верхнем углу окна под синей панелью выберите **Главная**, чтобы вернуться на домашнюю страницу портала Azure.

1. Выйдите и закройте все открытые вкладки браузера.
